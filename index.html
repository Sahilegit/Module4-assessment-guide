📚 Part 1: Understanding Performance Assessments

Core Concepts You Need to Know

Evaluation vs. Measurement

  • Evaluation = Judging quality based on comparison to a standard
  • Measurement = Determining how much of something exists

Assessment Types

Type Compares To Best For
Norm-referenced Other learners Academic settings, ranking
Criterion-referenced ✅ A fixed standard Workplace training, competency
Formative Ongoing during learning Feedback, improvement
Summative Final evaluation Certification, completion
💡 Workplace Insight

In corporate learning (like customer service training), you almost always want: Criterion-referenced + Formative feedback

Why? You need people to meet a specific performance standard, and you want to help them improve along the way—not compare them to each other.

✅ Knowledge Check 1: Concept Application

Scenario: You're designing a customer service training program for new hires at a tech company.

Question: Which assessment approach should you use?

Answer: B - Criterion-referenced + Formative

Why:

  • You need everyone to meet the same customer service standard (criterion-referenced)
  • You want to provide coaching and feedback during training so people can improve (formative)
  • Ranking employees against each other (norm-referenced) doesn't help them meet the required standard
  • Only testing at the end (summative only) misses opportunities to course-correct during learning

🎯 Part 2: What Makes a "Good" Assessment?

The Three Essential Qualities

Every assessment must be:

1. Reliable → Produces consistent results

  • Example: If I take the test twice, I get similar scores
  • Bad example: Test scores vary wildly based on which version someone gets

2. Valid → Measures what it's supposed to measure

  • Example: A typing test measures typing speed
  • Bad example: Multiple choice questions to assess hands-on repair skills

3. Fair → Equitable for all learners

  • Example: Clear instructions, accessible format, no bias
  • Bad example: Using complex academic language for frontline workers

📋 Part 3: Effective Evaluation Planning Framework

When designing any assessment, include these seven elements:

Element What It Means Example
Clarity Clear criteria for success "Score 85% or higher on customer handling scenarios"
Specificity Precise expectations "Demonstrate 3-step de-escalation process in role play"
Immediacy Fast feedback "Receive feedback within 24 hours"
Accessibility Discussion opportunities "Debrief session after assessment"
Affirmation Recognize effort "Acknowledge progress, not just final score"
Future-orientation Next steps "Here's what to practice next"
Justifiability Why it matters "This skill prevents customer churn"

🔬 Part 4: Matching Assessment Methods to Learning Domains

⚠️ The Critical Alignment Rule

Always align: Learning Objective → Learning Domain → Assessment Type

Assessment Methods by Domain

🧠 Cognitive Domain (Knowledge & Thinking)

Best for: Remembering facts, understanding concepts, analyzing, evaluating

Assessment Type When to Use Example
Multiple Choice Large-scale knowledge checks "Which of these is the correct refund policy?"
True/False Quick recall verification "Customers can return opened software: T/F"
Short Answer Quick recall with explanation "What are the 3 steps in our greeting protocol?"
Essay Deep thinking (why/how) "Analyze why this customer interaction failed"
Matching Relationships between concepts "Match each product feature to its benefit"

🛠️ Psychomotor Domain (Physical Skills)

Best for: Operating equipment, performing procedures, demonstrating techniques

Assessment Type When to Use Example
Checklist ✅ Yes/no behaviors "Did the agent open the correct system?" ☐ Yes ☐ No
Rating Scale ✅ Quality of performance "Navigation speed: 1=Slow, 5=Expert"
Performance Test ✅ Real-time demonstration "Complete a full ticket workflow while observed"
⚠️ CRITICAL:

Never use multiple choice questions to assess hands-on skills!

❤️ Affective Domain (Attitudes & Values)

Best for: Measuring engagement, confidence, professional identity

Assessment Type When to Use Example
Journals/Reflections Personal insights "Reflect on a challenging customer interaction"
Rating Scales Attitude measurement "How confident are you handling escalations? 1-5"
Discussions Group dialogue "What does customer-first service mean to you?"
Observations Behavioral indicators "Observing empathy in customer interactions"

🎓 Part 5: Choosing Instructional Methods

The Two Key Questions

Before selecting any instructional method, ask:

  1. Are learners gaining knowledge, skills, or attitudes?
  2. What should they be able to DO afterward?

Instructional Methods by Purpose

💡 Knowledge-Building Methods

Use when learners need to understand concepts, policies, or information

  • Lecturette (short presentation)
  • Readings
  • Large group discussion
  • Video
  • Case studies

Example: Teaching company policies → Use lecturette + reading

🛠️ Skill-Based Methods ⭐ Most valuable for workplace training

Use when learners need to perform tasks or demonstrate competencies

  • Role play - Practice interpersonal skills
  • Simulation - Safe environment to practice high-risk scenarios
  • Practice & feedback - Repetition with coaching
  • Demonstration - Show then do
  • On-the-job training (OJT) - Learn while working
The OJT Formula:
Tell → Show → Do → Feedback

Example: Teaching software navigation → Use demonstration + practice & feedback

🤝 Collaborative Learning Methods

Use when learners benefit from peer interaction and diverse perspectives

  • Small group discussion
  • Buzz groups (quick partner discussions)
  • Brainstorming
  • Fishbowl (inner circle discusses, outer observes)
  • Debate

Example: Exploring customer service scenarios → Use small group discussion

🔍 Reflective Learning Methods

Use when learners need to internalize experiences and build self-awareness

  • Journaling
  • Storytelling
  • Metaphor analysis

Example: Building empathy → Use storytelling + reflection

✅ Knowledge Check 3: Method Selection

For each objective, choose the BEST instructional method:

Objective 1: "Customer service agents will demonstrate the 5-step de-escalation process during difficult customer calls."
Objective 2: "Managers will understand the new remote work policy."
Objective 3: "Team members will value diverse perspectives in problem-solving."

Answers:

Objective 1: C - Role play

  • This is a skill (psychomotor) - they need to DO it
  • Role play lets them practice the behavior with coaching

Objective 2: B - Lecturette + reading + Q&A

  • This is knowledge (cognitive) - they need to understand information
  • Lecture and reading are efficient for information transfer

Objective 3: C - Small group discussion + reflection

  • This is an attitude (affective) - they need to value something
  • Discussion lets them experience diverse perspectives; reflection internalizes it

🚀 Part 6: Selecting Delivery Modalities

Step 1: What's the learning domain?
└─> Psychomotor (skill)? → Favor synchronous, hands-on methods

Step 2: How complex is it?
└─> Complex/High-risk? → Favor live instruction with feedback

Step 3: How big is your audience?
└─> Large + Geographically dispersed? → Consider asynchronous eLearning

Step 4: What resources do you have?
└─> Limited budget/time? → Asynchronous scales better

Synchronous (Real-Time) Modalities

Modality Best For Example
In-Person Classroom Complex skills, high interaction New hire onboarding with role plays
Virtual Classroom/Webinar Knowledge + discussion, dispersed teams Policy update webinar with Q&A
On-the-Job Training (OJT) Practical skills, immediate application Shadowing a senior agent

Pros: Immediate feedback, relationship building, real-time clarification
Cons: Scheduling challenges, travel costs (in-person), limited scalability

Asynchronous (Self-Paced) Modalities

Modality Best For Example
eLearning (interactive) Knowledge + some skills, large audiences Software tutorial with practice modules
Self-Study (readings, videos) Foundational knowledge Pre-work before live training
Social/Collaborative Platforms Peer learning, ongoing support Discussion forum for troubleshooting

Pros: Flexible timing, scales easily, cost-effective
Cons: Less immediate feedback, requires self-motivation, harder for complex skills

🏆 Final Challenge: Design Your Own Assessment Strategy

Scenario: You are an instructional designer at a customer support company. You need to design training for a new product launch.

Training Details:

  • Audience: 100 customer support agents (entry to mid-level)
  • Objective: "Agents will troubleshoot and resolve common customer issues with the new product, achieving 90% first-call resolution within 30 days."
  • Constraints:
    • Agents are across 3 time zones
    • Product launches in 4 weeks
    • Budget: $50,000
    • Must track competency achievement

Your Design Task

1. What is the primary learning domain?

2. What assessment method(s) will you use?

3. What instructional method(s) will you use?

4. What modality/modalities will you use?

5. How will you ensure quality? (Reliable, Valid, Fair)

6. How will you provide formative feedback?

Sample Solution (many approaches work!)

1. Primary Domain: Psychomotor (Skill)
The objective is about troubleshooting and resolving issues—this is a performance skill, not just knowledge.

2. Assessment Methods:

  • ✅ Performance-based simulation - Safe practice environment
  • ✅ Checklist during live calls - Verify competency
  • ✅ Case study analysis - Test troubleshooting logic

3. Instructional Method Sequence:

  1. eLearning modules - Product knowledge foundation
  2. Lecturette - Live product demo (30 min)
  3. Case study analysis - Small group troubleshooting
  4. Role play / Simulation - Practice scenarios
  5. On-the-job coaching - Supervised live calls

4. Modality Strategy:

  • Week 1-2 (Asynchronous): Self-paced eLearning + knowledge checks
  • Week 3 (Synchronous Virtual): Live demos + breakout case studies
  • Week 3-4 (OJT): Simulations + monitored calls with coaching
  • Ongoing: Discussion forum for peer support

5. Quality Assurance:

  • Reliability: Standardized checklist, trained coaches, consistent scenarios
  • Validity: Assessment mirrors real work (actual customer scenarios)
  • Fairness: Multiple time zones, mobile/desktop access, clear criteria

6. Formative Feedback:

  • Immediate feedback on eLearning knowledge checks
  • 3-2-1 feedback during role play (3 strengths, 2 improvements, 1 practice focus)
  • Real-time coaching during call monitoring
  • Daily 10-minute debrief sessions
  • Weekly progress tracker with checklist scores

Summative Assessment (30 days): Monitor 5 live calls with 90% FCR requirement

📚 Key Takeaways

The Non-Negotiables

  1. ✅ Always use criterion-referenced + formative assessment in workplace training
  2. ✅ Never use MCQs to assess hands-on skills (psychomotor domain)
  3. ✅ Always align: Objective → Domain → Assessment → Method → Modality
  4. ✅ Every assessment must be: Reliable, Valid, Fair
  5. ✅ Skill-based training needs: Practice + Feedback + Observation
